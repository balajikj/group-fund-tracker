üìã Step 1: Firebase Setup (5 minutes)

1.1 Create Firebase Project
  • Go to Firebase Console
  • Click "Add project" or "Create a project"
  • Name it (e.g., "group-fund-tracker")
  • Disable Google Analytics (not needed)
  • Click "Create project"
1.2 Enable Firestore Database
  • In left menu: Build ‚Üí Firestore Database
  • Click "Create database"
  • Select "Start in production mode"
  • Choose a location (closest to you)
  • Click "Enable"
1.3 Set Up Security Rules
  • In Firestore, click "Rules" tab
  • Copy content from firestore.rules file
  • Paste and click "Publish"
1.4 Enable Authentication
  • In left menu: Build ‚Üí Authentication
  • Click "Get started"
  • Click "Email/Password"
  • Enable "Email/Password" (first toggle)
  • Click "Save"
1.5 Get Your Configuration
  • Click the ‚öôÔ∏è gear icon ‚Üí Project settings
  • Scroll to "Your apps" section
  • Click the web icon </>
  • Give it a nickname (e.g., "fund-tracker")
  • Click "Register app"
  • Copy the firebaseConfig object
  • Paste it into firebase-config.js (replace placeholder)

üë§ Step 2: Create Admin User (2 minutes)

2.1 Create User in Authentication
  • Go to Authentication ‚Üí Users tab
  • Click "Add user"
  • Enter email and password
  • Click "Add user"
  • Copy the User UID (long string)
2.2 Add User to Firestore
  • Go to Firestore Database ‚Üí Data tab
  • Click "Start collection"
  • Collection ID: members
  • Document ID: Paste the User UID
  • Add fields:
    • name (string): Your Name
    • role (string): Admin
    • lifetimeContribution (number): 0
  • Click "Save"

üåê Step 3: Deploy to GitHub Pages (8 minutes)

3.1 Create GitHub Repository
  • Go to GitHub - New Repository
  • Repository name: group-fund-tracker
  • Make it Public
  • Don't initialize with README
  • Click "Create repository"
3.2 Push Your Code

Open terminal/command prompt in your project folder and run:

git init git add . git commit -m "Initial commit" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/group-fund-tracker.git git push -u origin main

Replace YOUR_USERNAME with your GitHub username

3.3 Enable GitHub Pages
  • In your repository, go to Settings
  • Scroll to "Pages" in left menu
  • Source: Deploy from a branch
  • Branch: main / (root)
  • Click "Save"
  • Wait 1-2 minutes for deployment
  • Your URL: https://YOUR_USERNAME.github.io/group-fund-tracker/
3.4 Authorize Domain in Firebase
  • Back to Firebase Console
  • Authentication ‚Üí Settings ‚Üí Authorized domains
  • Click "Add domain"
  • Enter: YOUR_USERNAME.github.io
  • Click "Add"

‚úÖ Step 4: Test Your Application (5 minutes)

  • Open your GitHub Pages URL
  • Login with the Admin credentials you created
  • You should see the dashboard!
  • Try adding a test contribution
  • Check that the fund balance updates

üÜò Need Help?

Common Issues:
  • Login fails: Check that you replaced Firebase config in firebase-config.js
  • Permission denied: Verify Firestore rules are published and user exists in members collection
  • Page not loading: Wait a few minutes after enabling GitHub Pages
  • Can't see admin panel: Make sure role is "Admin" or "CoAdmin" (case-sensitive)
